# FITS fixture recipes — Inbox & Plans verification scenarios

Shared fixture recipes for every scenario under `041-inbox-plan-surface/`,
`025-filesystem-plan-application/`, `audit-log/`, and
`journeys/grand-inbox-journey/`. Runner mechanics (launch, reset, bridge
connection) live in `../AGENT-RUNNER.md` — this file only covers **test data**.

All fixtures are tiny synthetic FITS files (4×4 pixels, ~6 KB each) with
fully controlled headers, generated by the `make_fits.py` helper below. They
mimic N.I.N.A.-style headers, matching the keyword conventions the metadata
extractor reads (see `docs/development/077-fits-header-analysis.md` and
`scripts/gen-mock-fits.py`, the repo's canonical generator this helper is
derived from).

## The helper: `make_fits.py`

Write this file to `C:\dev\astro-plan\test-data\make_fits.py` on the Windows
machine (from WSL: `/mnt/c/dev/astro-plan/test-data/make_fits.py`). It is
self-contained (stdlib only; Python 3.10+, `py` or `python` on Windows both
work).

```python
#!/usr/bin/env python3
"""make_fits.py <out.fits> KEY=VALUE [KEY=VALUE ...]

Writes a minimal valid FITS file (4x4 uint16 image) with the given header
cards. VALUE is auto-typed: int, float, T/F bool, else string. A VALUE of
`-` omits the card entirely (lets recipes show intent for absent keywords).
"""
import struct, sys

BLOCK = 2880

def card(kw, val):
    kw = kw.upper().ljust(8)[:8]
    if isinstance(val, bool):
        s = f"{kw}= {'T' if val else 'F':>20}"
    elif isinstance(val, int):
        s = f"{kw}= {val:>20}"
    elif isinstance(val, float):
        s = f"{kw}= {val:>20.6f}"
    else:
        s = f"{kw}= '{str(val):<18}'"
    return s.ljust(80)[:80].encode("ascii")

def autotype(raw):
    if raw in ("T", "F"):
        return raw == "T"
    try:
        return int(raw)
    except ValueError:
        pass
    try:
        return float(raw)
    except ValueError:
        return raw

def main():
    out, pairs = sys.argv[1], sys.argv[2:]
    cards = [
        card("SIMPLE", True), card("BITPIX", 16), card("NAXIS", 2),
        card("NAXIS1", 4), card("NAXIS2", 4), card("BZERO", 32768),
        card("BSCALE", 1),
    ]
    for p in pairs:
        k, _, v = p.partition("=")
        if v == "-":
            continue
        cards.append(card(k, autotype(v)))
    cards.append(b"END".ljust(80))
    header = b"".join(cards)
    header += b" " * (BLOCK - len(header) % BLOCK)
    data = struct.pack(">16h", *([100] * 16))
    data += b"\x00" * (BLOCK - len(data) % BLOCK)
    with open(out, "wb") as f:
        f.write(header + data)
    print(f"wrote {out}")

if __name__ == "__main__":
    main()
```

## Common card sets

Every generated file gets these baseline cards (append to each command):

```
INSTRUME="ZWO ASI2600MM Pro" TELESCOP="Celestron C925" FOCALLEN=1645.0
GAIN=100 OFFSET=50 SET-TEMP=-10.0 CCD-TEMP=-10.1 XBINNING=1 YBINNING=1
XPIXSZ=3.76 YPIXSZ=3.76 SWCREATE="N.I.N.A. 3.2.0.9001 (x64)" EQUINOX=2000.0
```

Shorthand used below: `<COMMON>` = the card list above, written as
`KEY=VALUE` arguments. In PowerShell, quote arguments containing spaces:
`'INSTRUME=ZWO ASI2600MM Pro'`.

The light-frame target used throughout is **NGC 7000** (RA 313.0 / DEC 44.5
decimal degrees) — a prominent catalog object so coordinate-based target
resolution (spec 041 FR-052) has a deterministic nearest match in the bundled
seed catalog.

## Recipes

Base folder for all recipes: `C:\dev\astro-plan\test-data\`. Each recipe is
a PowerShell block run in the Windows checkout (`cd C:\dev\astro-plan`).
`$MF = 'python test-data\make_fits.py'` — invoke as `Invoke-Expression
"$MF <args>"` or just type the full command. Recipes assume the containing
folder is (or is inside) a registered **inbox** root unless stated otherwise.

### RECIPE-MIXED — one leaf folder, three homogeneous groups

Purpose: US10/FR-034/SC-012 (mixed folder → N single-type sub-items),
US13/FR-043 (source-group provenance), US6/FR-021 (stats).

Folder `inbox-drop\night1` containing:
- 2 lights, Ha, 300 s (NGC 7000)
- 2 lights, Ha, 120 s (NGC 7000) — same type+filter, different exposure
- 2 darks, 300 s

```powershell
New-Item -ItemType Directory -Force -Path 'C:\dev\astro-plan\test-data\inbox-drop\night1'
cd 'C:\dev\astro-plan\test-data\inbox-drop\night1'
foreach ($i in 1,2) {
  python C:\dev\astro-plan\test-data\make_fits.py "light_ha_300_000$i.fits" `
    IMAGETYP=LIGHT 'OBJECT=NGC 7000' FILTER=Ha EXPTIME=300.0 EXPOSURE=300.0 `
    "DATE-OBS=2026-06-2${i}T22:0${i}:00.000" RA=313.0 DEC=44.5 <COMMON>
  python C:\dev\astro-plan\test-data\make_fits.py "light_ha_120_000$i.fits" `
    IMAGETYP=LIGHT 'OBJECT=NGC 7000' FILTER=Ha EXPTIME=120.0 EXPOSURE=120.0 `
    "DATE-OBS=2026-06-2${i}T23:0${i}:00.000" RA=313.0 DEC=44.5 <COMMON>
  python C:\dev\astro-plan\test-data\make_fits.py "dark_300_000$i.fits" `
    IMAGETYP=DARK EXPTIME=300.0 EXPOSURE=300.0 `
    "DATE-OBS=2026-06-2${i}T12:00:00.000" <COMMON>
}
```

Expected classification result: **3 single-type sub-items**, zero "mixed"
items: `light · Ha · 300s` (2 files), `light · Ha · 120s` (2 files),
`dark · 300s` (2 files).

## On-disk assertion helpers

Scenarios assert real file locations after Apply. From WSL (read-only,
against the Windows filesystem):

```bash
find /mnt/c/dev/astro-plan/test-data/library-lights -name '*.fits' | sort
find /mnt/c/dev/astro-plan/test-data/inbox-drop -name '*.fits' | sort
sha256sum /mnt/c/dev/astro-plan/test-data/organized-lights/**/*.fits
```

Byte-stability (SC-016, overrides never touch files): capture `sha256sum`
before an override, compare after — hashes must be identical.
