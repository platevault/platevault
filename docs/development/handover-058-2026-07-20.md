## What the feature does now

Scanning a folder writes an `inbox_source_groups` row and **no inbox item**. The
folder appears in the Inbox as a source-group row that is structurally
non-confirmable (no `inboxItemId` exists to hand `inbox.confirm`), carrying one
action: **Classify**. Classifying materialises the folder's real single-type
items, the group row disappears, and selection follows CHK011.

The hidden placeholder row this spec exists to delete is gone, along with the
read-side suppression that existed only to hide it.

## What is NOT done

### 1. Phase 6b — the needs-review split (T050–T052). Real feature work.

**Probed, not assumed.** Resolving a needs-review item's two files into `light`
and `dark` applies both overrides, reports `unclassified`, and materialises
**zero** siblings.

The mechanism is documented in Q-9 and still holds: `reclassify_v2` writes
`manual_override`, but the subsequent `materialize_sub_items` →
`seed_sub_item_cache` rebuild deletes and re-inserts that item's evidence with
`manual_override: None`. The re-derivation therefore sees an empty type set. The
fix is to make the user's overrides survive into materialisation — that is the
whole of T050, and it is delicate because that rebuild is what keeps a later
`inbox.classify(sub_id)` a cache hit rather than a silent re-derive.

A **characterization test** now pins the gap in-suite:
`t050_gap_two_frame_types_resolve_without_splitting` (`reclassify.rs`). It
asserts *current* behaviour. When T050 lands it **must be rewritten** to assert
two siblings — its failure then is the intended signal, not a regression.

## Traps worth not re-learning

- **Green tests proved nothing for T012.** Deleting the placeholder left all
  2528 workspace tests passing — `inbox_scan_folder` had *zero* Layer-1 coverage
  of that behaviour. Two L1 tests were added; L3 remains the only end-to-end
  check.
- **A vacuous assertion nearly shipped.** "No row from `list_inbox_sub_items` has
  an empty `group_key`" cannot fail — that query filters `group_key != ''` in
  SQL. Compare the *unfiltered* count instead.
- **testid prefix collision.** `inbox-source-group-<id>` is a strict prefix of
  `inbox-source-group-classify-<id>`, and `testid_suffix` returns the first
  match. Derive ids from the button prefix.
- **Never chain a gate and a commit in one command.** `clippy | grep -c` piped
  into a `git commit` let a `-D warnings` failure ship (`b1348161`), because
  `cargo check` and `nextest` stay green on an unused import.
- **`-p app_core` is not `-p app_core_inbox`.** Targeted runs against the wrong
  crate reported 480 passing tests that never included the code under change.
