## CI gate

Pre-merge: `just check` runs lint + test + typecheck + `check-generated`.
`check-generated` regenerates the Rust→JSON contract schemas and the
tauri-specta TS bindings, then fails if either is out of sync with the tree.

## Where to start when something breaks

- **Transition refused unexpectedly** — first look at
  `crates/app/core/src/transition_use_case.rs::apply_transition`'s decision
  order (noop → edge validity → actor=system → plan-required). The error
  envelope's `code` field tells you which gate refused.
- **Binding compiles but call fails at runtime** — check that the command
  is registered in `apps/desktop/src-tauri/src/lib.rs::specta_builder()`
  inside `collect_commands![...]` and that the capability
  (`apps/desktop/src-tauri/capabilities/default.json`) grants it.
- **Schema mismatch after editing a contract DTO** — run
  `cargo run -p contracts_core --bin generate-contracts` then
  `cargo test -p desktop_shell --test bindings`; commit the diff.
