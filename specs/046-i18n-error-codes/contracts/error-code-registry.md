# Contract: Error-Code Registry & Translation

**Feature**: `046-i18n-error-codes`

This is the language-neutral contract per Constitution principle V. It already
exists (spec 042 T011); this feature formalizes its consumption.

## 1. Authoritative set (backend)

`crates/contracts/core/src/error_code.rs`:

```rust
#[derive(Clone, Copy, Debug, PartialEq, Eq, Serialize, Deserialize, Type)]
pub enum ErrorCode {
    #[serde(rename = "path.not_exists")]   PathNotExists,
    #[serde(rename = "project.not_found")] ProjectNotFound,
    // … ~95 variants, each with an explicit byte-exact wire rename …
    #[serde(rename = "internal.error")]    InternalError, // generic wrap
}
```

Wire strings are byte-exact via explicit `#[serde(rename)]` (dots survive).

## 3. Error envelope (transport — unchanged)

```ts
interface ContractError {
  code: ErrorCode;          // ← tightened from `string` in this feature
  message: string;          // backend diagnostic text; NOT shown raw to users
  details?: unknown;        // optional structured payload (e.g. { dimensions })
}
```

The transport shape `{ code, message, details? }` is preserved (assumption in
spec). Only the *type* of `code` tightens and how `message` is *derived for
display* changes (via the catalog), not the wire contract.

## 4. Translation map (frontend, single point)

`apps/desktop/src/lib/error-messages.ts` + `apps/desktop/src/lib/errors.ts`:

```ts
// code → catalog message fn (values are catalog keys, not literals)
export const ERROR_MESSAGES: Partial<Record<ErrorCode, () => string>> = {
  'path.not_exists': m.err_path_not_exists,
  'project.not_found': m.err_project_not_found,
  // …
};

export function errMessage(err: unknown): string {
  if (isContractError(err)) {
    const fn = ERROR_MESSAGES[err.code];
    if (fn) return fn();
    logUnmappedCode(err.code);              // FR-010 / FR-011
    return m.err_generic_fallback();        // safe fallback, never the raw code
  }
  if (err instanceof Error) return err.message;
  return String(err);
}
```

## 5. Contract guarantees (mapped to FR/SC)

| Guarantee | Mechanism | FR / SC |
|-----------|-----------|---------|
| One code set, no drift | Rust enum → generated TS union, CI diff | FR-006, FR-007, SC-004 |
| Adding a code → frontend gap visible pre-release | `Record<ErrorCode, …>` exhaustiveness / union-iterating test | FR-007, SC-003 |
| One translation point | only `errMessage()` turns errors into text | FR-008, SC-004 |
| No raw code/exception to user | fallback returns catalog text only | FR-009, SC-005 |
| Unmapped code logged | `logUnmappedCode()` at the single point | FR-010, FR-011, SC-005 |
| Friendly message per code | catalog entry `err_<code>` | FR-008, SC-005 |

## 6. Versioning

Additive: new codes append enum variants (MINOR). Removing/renaming a code is a
breaking contract change — the generated union changes, the CI diff trips, and the
frontend map's exhaustiveness check flags the gap before release.
