# Spec 062 Dual-Lens Critique

**Date**: 2026-07-21
**Scope**: Product and engineering readiness before implementation-task creation
**Verdict**: GO

## Executive Summary

The immutable-session, topology, project, calibration, correction, security,
and scale architecture is sound. Independent product and engineering critics
found missing executable contract paths plus persistence, recovery, pagination,
authorization, and scale-proof gaps. These are completion defects in already
approved behavior, not reasons to reopen the owner's product model.

All must-address items were remediated. Fresh product, contract/test,
SQLite/data, security, checklist, and cross-artifact reviews returned GO.

## Product Lens Findings

### Problem and user value

- Manual grouping is promised when geometry is unavailable, but no command can
  create the first manual proposal or reviewed cross-target association.
- Flat organization and cross-night candidate selection are promised, while the
  reviewed handoff contract names only dark and bias.
- Observing-night identity depends on a confirmed site timezone, but the user
  resolution flow for site selection and timestamp conflict is absent.

### User experience and accessibility

- Yellow/red warning semantics rely on color names without requiring textual
  severity, focus behavior, or screen-reader announcements.
- Long-running topology and materialization progress lacks explicit accessible
  status and cancellation criteria.

### Success measurement

- Correctness and latency are measurable, but review comprehension and workflow
  completion are not. A small moderated evidence-comprehension criterion would
  detect a technically correct but unusable review surface.

## Engineering Lens Findings

### Architecture and contracts

- Cross-target association creation is circular: acceptance requires an
  association that no command can create.
- The primary Inbox materialization mutation has no portable contract for
  approval digest, operation subtype, paginated outputs, retry, audit, events,
  or failures.
- Supersession, lineage, and reclassification exact collections remain embedded
  in 500-item DTOs despite larger supported data.

### Failure modes and resource bounds

- Update View caps sessions but not plan items or frames. Complete preflight
  needs persisted chunking and a finite per-operation work ceiling with a typed
  continuation or refusal.

## Cross-Lens Insights

- A first-class manual proposal command solves both the missing-geometry UX and
  the cross-target association ownership cycle.
- A portable Inbox mutation contract makes the P1 user journey testable and
  gives command fencing, audit, and pagination one consistent boundary.
- Count-plus-pagination result snapshots reduce payload risk while preserving
  the complete history users need to review.
- Explicit non-color warning semantics improve both accessibility and the
  reliability of automated UI acceptance evidence.

## Proposed Artifact Updates

1. Extend session/group contracts with manual proposal creation and atomic
   reviewed cross-target association creation.
2. Add an Inbox materialization contract to the contract index.
3. Replace exact inline supersession, lineage, and reclassification collections
   with counts and paginated child queries.
4. Extend calibration candidate/handoff contracts to flats.
5. Add acquisition-site selection and timezone-conflict resolution contracts.
6. Add explicit accessibility and review-comprehension requirements.
7. Add persisted Update View work chunking and resource ceilings.

After remediation, re-run product, engineering, architecture, database, and
security review before creating implementation Beads.

## Remediation Applied

The artifact set now includes the proposed updates and the later engineering
corrections: portable Inbox/site-resolution contracts; manual reviewed relation
creation; flat handoff; one-to-many project replacement; complete immutable plan
and apply result snapshots; destination topology and lineage; deterministic
watermarked pagination; typed safe errors and redacted projections; bounded
Update View chunks; prepared install intents and fenced recovery; authorization;
outward RTree quantization; sentinel CTE limits; cancellable traversal previews;
reproducible scale evidence; exact released upstream gates; local diagnostics;
and correction-overlay previews.
