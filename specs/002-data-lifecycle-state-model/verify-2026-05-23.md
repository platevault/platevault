## actor=system edge policy

Verified at `transition_use_case.rs::touches_blocked` + lines 102-112 + 280-290 (preview). System actor is rejected with `ActorNotAuthorised` on any edge that does not touch `blocked`. Test `rejects_system_on_non_blocked_edge` confirms. `allows_system_on_blocked_recovery` confirms the recovery edge is permitted.

Note: data-model.md §AuditLogEntry Invariants says system-actor refusals MUST be audit-logged. The current `apply_transition` returns the refusal envelope but does NOT write a refused-outcome audit row before returning (the early-return short-circuits before any DB write). This is a minor deviation from the "audit-logged" wording — the refusal envelope is returned to the caller but no `outcome=refused` row is persisted. Tasks.md T038 calls this out as a requirement but the code path skips it. **Minor gap — not enough to block close, but should be tracked.**

## status: "noop" semantics

Verified. Same-state writes:

`TransitionResponse::noop` omits `applied_at, prior_state, new_state, audit_id, plan_id, error` — matches the contract `if status=noop then not anyOf(requires audit_id, requires error)` allOf clause.

## Contract envelope (provenance.unreviewed)

Verified. `lifecycle.transition.json` `ErrorEnvelope.details.blockingFields[]` schema = `{fieldPath: string, requiredOrigin: "reviewed"}`. Rust DTO `TransitionErrorCode::ProvenanceUnreviewed` (lifecycle.rs:292) `#[serde(rename_all = "snake_case")]` → `"provenance_unreviewed"`.

**Schema vs Rust serde divergence**: the JSON schema enumerates `provenance.unreviewed` (with a dot) but the Rust enum serializes as `provenance_unreviewed` (snake_case). This is the same `_` vs `.` divergence that already exists for `transition.refused` / `plan.required` etc., and is consistent across all error codes in the codebase. **This is a pre-existing divergence between contract source-of-truth (dot-separated) and the Rust serde repr (underscore).** Tests in `packages/contracts/tests/lifecycle.transition.errors.test.ts` (T030) should be inspected for fixture shape — if they assert the dot form, the Rust→TS adapter must be doing the conversion. Worth flagging but not a spec-002 regression.

Body shape: `details.blockingFields` (camelCase) populated at `transition_use_case.rs:233-237` matches schema verbatim.

## Findings By Severity

### Must Fix Before Proceeding

None. Spec 002 implementation matches the spec.

### Should Address

1. **Refused-outcome audit row not written for system-actor refusals.** `transition_use_case.rs::apply_transition` returns the refusal envelope without first writing an `outcome=refused, actor=system` audit row. data-model.md §AuditLogEntry Invariants requires "the rejection itself is audit-logged as `actor == system, outcome == refused`". The current path returns the response but skips the audit-write. The transactional refusal-logging path for `actor=user` refusals shares the same gap — the early returns bypass the audit table.
   - Impact: low — refusals are still observable via the response envelope and the `useRefusals` projection, but they are not persisted across restarts.
   - Recommendation: open a follow-up to add a refused-outcome audit write before each early return. Possibly out-of-scope for spec 002 closure if explicitly deferred; otherwise this is a real gap.

2. **Contract code form `provenance.unreviewed` (dot) vs serde `provenance_unreviewed` (snake_case).** Pre-existing divergence shared across all error codes — fixture tests under `packages/contracts/tests/` are the source of truth for which form the wire uses. Not a spec-002 regression but worth a one-time alignment pass.

### Notes

- `LifecycleRepository::field_origins` is correctly implemented for SQLite + the in-memory stub (returns empty map, correctly forcing the gate to fire since `Reviewed` is never present in the stub map).
- Alias coverage in `provenance_asset_type()` (lifecycle.rs:438-445) ensures the gate fires whether the request was routed as `inventory_session` (contract variant) or `acquisition_session` (storage table) — matches the documented alias requirement.
- `transition_preview` (use case T039) is read-only and does NOT enforce the action-bound review gate (no `check_action_review` call in preview). This is defensible for a UI dry-run but inconsistent with `apply_transition`. If the UI uses preview to decide whether to enable a confirm button, the user will see an enabled button that then refuses on apply with `provenance.unreviewed`. Worth aligning, but spec does not mandate it.
- Test coverage looks comprehensive: unit tests cover edge tables (T026/T027/T028/T035a), integration tests cover transactional behavior (T029/T031), contract fixtures cover error codes (T030), and Playwright covers the UI write-side seam (T043).
