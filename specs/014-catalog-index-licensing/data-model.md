# Data Model: Catalog Index Licensing

**Spec**: 014-catalog-index-licensing | **Plan**: [plan.md](./plan.md)

This data model covers the registry and attribution surface only. The
minimal catalog entry shape (`name, identifiers, ra, dec, source`) is
owned by spec 013 and is referenced but not redefined here.

> **`ProvenancedValue` carve-out (R-3.2)**: Catalog entries are
> app-owned reference data, NOT user assets. Fields on `Catalog` and
> `LicenseAttribution` do NOT use `ProvenancedValue<T>`. The catalog as
> a whole carries source provenance via the manifest + `LicenseAttribution`.
> User-visible Target identity that diverges from catalog data lives in
> spec 013 with full provenance tracking.

## Entities

### Catalog

Represents one registered catalog index known to the app.

| Field          | Type                       | Notes                                                   |
| -------------- | -------------------------- | ------------------------------------------------------- |
| `id`           | `string` (slug)            | Stable identifier, e.g. `messier`, `ngc`, `ic`.         |
| `name`         | `string`                   | Human-readable display name.                            |
| `version`      | `string` (semver-ish)      | Bundle version. Bumped by CI on every regeneration.     |
| `license`      | `string` (short code)      | e.g. `public-domain`, `cc-by-4.0`, `apache-2.0`.        |
| `origin`       | `enum`                     | `built_in` (reserved, unused in v1), `downloaded` (all v1 catalogs), or `user` (reserved for v1.x). (R-1.3, R-3.3) |
| `source_url`   | `string` (URI)             | Upstream source the bundle was generated from.          |
| `last_updated` | `Timestamp` (RFC 3339 UTC) | When the bundle was regenerated by CI (or registered).  |
| `entry_count`  | `integer` (>= 0, optional) | Number of entries in the index. Surfaced in Settings.   |

Invariants:

- `id` is unique across the registry.
- `downloaded` catalogs are installed from the project-hosted manifest
  (all v1 catalogs). They can be re-downloaded but not removed at
  runtime except via an explicit admin action.
- `built_in` is reserved for forward-compat (future emergency-fallback);
  zero `built_in` catalogs exist in v1. (R-3.3)
- `user` is reserved for v1.x. The backend MUST reject any operation
  with `origin = "user"` with error code `origin.not_implemented` in v1.
  (A2, R-1.3)
- Every `Catalog` MUST have at least one matching `LicenseAttribution`
  record (even if the license is `public-domain`).

### LicenseAttribution

Represents the per-catalog notice surface used in Settings and in
generated NOTICE files for downstream redistribution. (R-2.2)

| Field                  | Type                       | Notes                                                  |
| ---------------------- | -------------------------- | ------------------------------------------------------ |
| `catalog_id`           | `string` (slug)            | FK to `Catalog.id`.                                    |
| `license`              | `LicenseShortCode`         | Closed enum (R5). Echoes `Catalog.license`.            |
| `text`                 | `string` (non-empty)       | Full required notice text, verbatim.                   |
| `link`                 | `string` (URI)             | Source link the notice references.                     |
| `accessed_on`          | `Timestamp` (optional)     | Date the source was fetched. Used for public-domain    |
|                        |                            | "verified on" entries.                                 |
| `author`               | `string` (optional)        | **REQUIRED** if `license` matches `cc-by-*` or        |
|                        |                            | `cc-by-sa-*`. Author or rights-holder name.            |
| `title`                | `string` (optional)        | **REQUIRED** if `license` matches `cc-by-*` or        |
|                        |                            | `cc-by-sa-*`. Work title.                              |
| `license_uri`          | `string` (URI, optional)   | **REQUIRED** if `license` matches `cc-by-*` or        |
|                        |                            | `cc-by-sa-*`. Canonical license URL.                   |
| `modifications_notice` | `string` (optional)        | Required if the project has modified the upstream      |
|                        |                            | data (e.g. field subset, coordinate normalisation).    |
|                        |                            | Describes the nature of modifications.                 |

Invariants:

- `text` is never empty. Public-domain catalogs still carry a
  "verified: <source>, accessed <date>" string so users can audit the
  declaration.
- For `license` values matching `cc-by-4.0` or `cc-by-sa-4.0`,
  `author`, `title`, and `license_uri` are REQUIRED. The data model
  enforces this via a conditional schema constraint.
- `modifications_notice` is REQUIRED if the catalog data has been
  transformed by the project (e.g. column subsetting, normalisation).
  It is OPTIONAL (and typically absent) for unmodified redistribution.
- `link` MUST resolve to a stable source URL or an archived snapshot
  (e.g. Internet Archive). Mirror lists are out of scope for v1.
- One `Catalog` MAY have multiple `LicenseAttribution` rows when a
  bundle aggregates more than one upstream source; the panel renders
  them grouped under the catalog.

## Relationships

```
Catalog 1 ───< LicenseAttribution  (by catalog_id)
```

No other entity references either record in v1. Spec 013's target
lookup pipeline reads catalog index files directly via the registry
and does not embed catalog metadata into target records.

## Storage

- Downloaded catalogs: SQLite table `catalog_downloaded(id, name,
  version, license, source_url, last_updated, entry_count)` plus a
  paired `catalog_downloaded_attribution(catalog_id, license, text,
  link, accessed_on, author, title, license_uri,
  modifications_notice)` table. All thirteen v1 catalogs use this
  path. (A2 — `catalog_user*` tables removed; user-added deferred)
- `built_in` origin: no storage in v1; enum reserved for forward-compat.
- `user` origin: deferred to v1.x; no tables in v1.
- The `Catalog` and `LicenseAttribution` records returned by the
  contracts are read from the `catalog_downloaded` tables in v1.

## Out of Scope

- Catalog entries themselves (owned by spec 013).
- Full user-facing update UI (deferred to v1.x; see research R3).
- User-added catalogs (`origin = "user"`) — deferred to v1.x. (A2)
- Cross-catalog identifier reconciliation — owned by spec 013 (OpenNGC
  is the canonical NGC+IC source).
